## 1. 产品概述

阿里云数据管理服务（Data Management Service, DMS）是一款支撑数据库全生命周期的一站式数据管理服务，覆盖数据研发、数据安全、数据运维、数据资产、性能优化五大核心领域，为企业提供安全、高效、规范的数据库DevOps解决方案。

### 1.1 产品设计理念（基于阿里云DMS官方）

#### 🎯 「一个工作台」设计原则
- **场景整合而非功能堆砌**：围绕用户真实工作流组织功能，而非按技术领域切割菜单
- **闭环体验而非断点跳转**：从「设计 → 预览 → 审批 → 执行 → 验证」完整闭环，无需在多页面间跳转
- **上下文感知智能推荐**：在对的场景给出对的建议（比如设计表时提示索引优化建议）

#### 🔄 DDL变更全链路整合
```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  可视化设计  │ ──▶ │  SQL预览    │ ──▶ │  审批流程   │ ──▶ │  执行与监控  │
│  (改表/建表) │     │  (风险提示)  │     │  (多级审批)  │     │  (进度追踪)  │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
                                                               │
                                                               ▼
                                                         ┌─────────────┐
                                                         │  结果验证   │
                                                         │  (回滚预案)  │
                                                         └─────────────┘
```
> **核心原则：用户在同一个工作台中完成所有DDL变更操作，无需切换页面**

### 1.2 核心价值
- **安全可控**: 细粒度权限控制、敏感数据脱敏、全链路操作审计
- **高效协作**: 标准化研发流程、多人协作、版本化管理
- **智能优化**: SQL智能审核、性能诊断、优化建议
- **统一管理**: 支持20+种数据库类型，统一管理体验

## 2. 核心功能模块

### 2.1 数据开发模块

#### 2.1.1 SQL查询窗口
- **基础功能**
  - SQL编辑器：语法高亮、智能补全、格式化、代码折叠
  - 多Tab页支持，并行查询多库多表
  - 查询结果集分页展示、导出（Excel/CSV/JSON）
  - 结果集编辑：支持直接修改并提交到数据库
  - 执行计划查看与分析
  - SQL执行历史记录与回放
  - 常用SQL模板收藏与管理
  
- **高级功能**
  - 跨库查询（支持异构数据库关联查询）
  - 大结果集流式导出
  - 查询结果可视化图表
  - SQL参数化执行
  - 多行数据批量编辑

#### 2.1.2 数据变更管理
- **普通数据变更**
  - 支持INSERT/UPDATE/DELETE/REPLACE语句
  - 变更前数据预览与影响行数预估

- **无锁结构变更（Online DDL）**

  - 变更暂停/终止/回滚控制

#### 2.1.3 数据导入导出
- **数据导入**

  - 大文件分片上传
  - 字段映射配置
  - 导入速度与并发控制
  - 导入进度与错误日志
  
- **数据导出**
  - 全量/条件导出
  - 定时导出任务
  - 导出文件加密
  - 导出权限审批流程

### 2.2 结构设计与DDL变更工作台（核心整合模块）

> **基于阿里云DMS设计理念：将表结构设计与DDL部署流水线整合为统一工作台，避免用户在多个功能间跳转**

#### 2.2.1 DDL变更一体化工作台（核心）
- **统一入口体验**
  - 单一入口完成：选择实例→选择库→选择表→可视化修改→预览SQL→提交审批→执行→验证
  - 左侧：表列表与结构树（库→表→字段三级导航）
  - 右侧：可视化设计面板与SQL预览区
  - 底部：变更日志与执行状态
  - **整个DDL变更流程在同一个页面内闭环完成**

- **上下文智能感知**
  - 选择实例后，自动加载该实例下的所有数据库Schema
  - 选择表后，自动加载当前结构并进入"修改模式"
  - 修改字段时，实时显示变更类型（ADD/MODIFY）与SQL预览
  - 自动识别高危变更（删表、删字段、字段类型缩窄）并标红提醒

#### 2.2.2 可视化表结构设计
- **表结构管理**
  - 可视化建表/改表一体化界面（同一个设计器支持新建与修改）
  - 字段类型、长度、默认值、注释、主键、索引配置
  - 外键、触发器、存储过程可视化管理
  - 表结构对比工具（历史版本对比、环境对比）
  - 结构变更历史追踪与回滚点
  - **直接从数据库导入表结构进行修改，无需手动输入**

- **数据库设计**
  - ER图可视化设计
  - 批量表结构导入导出
  - 设计规范自动检查（字段命名、索引规范、注释完整性）
  - 设计文档自动生成

#### 2.2.3 Online DDL无锁变更（深度整合）
- **变更策略自动选择**
  - 智能判断：根据表大小、DDL类型、业务高峰自动选择最优变更策略
  - 策略选项：原生Online DDL、PT-OSC、原生Copy、无锁变更
  - 变更前风险评估：预计时长、锁表风险、业务影响等级

- **执行监控与控制**
  - 实时进度展示（百分比、已处理行数、剩余时间）
  - 暂停/继续/终止控制（阿里云DMS独有"安全暂停"能力）
  - 速度档位调节（低峰全速/高峰限流/维护窗口模式）
  - 异常自动回滚与熔断保护

- **变更后自动验证**
  - 结构一致性校验（源表 vs 目标表）
  - 数据完整性校验（行数、抽样校验）
  - 性能回归检测（变更前后QPS对比）
  - 自动生成变更验证报告

#### 2.2.4 结构变更一键提交流程
- **设计→审批无缝衔接**
  - 在设计器中完成表结构修改后，一键提交工单
  - 自动带入：变更SQL、目标库、变更类型、影响范围
  - 自动计算：影响行数、预计执行时间、风险等级
  - 无需离开设计页面，弹窗完成工单信息填写

- **审批上下文透传**
  - 审批人可直接查看：可视化变更前后对比图
  - 支持：字段级差异高亮、SQL预览、影响分析报告
  - 审批意见可直接@给提交人进行修改

#### 2.2.5 数据库版本管理
- **SQL脚本版本控制**
  - Git/SVN集成
  - 变更脚本自动生成
  - 版本分支管理
  - 环境差异对比
  - 多环境一键发布

### 2.4 流程与协作模块

#### 2.4.0 设计-审批-执行链路打通（关键体验优化）
> **阿里云DMS核心理念：工具嵌入流程，而非流程割裂工具**

- **从设计器直接发起审批**
  - 用户在表结构设计器中完成修改后，点击「提交变更」即可发起审批
  - 无需：复制SQL→打开工单页面→粘贴SQL→重新选择数据库
  - 所有上下文信息（数据库、表、变更SQL、风险评估）自动带入

- **审批页面复用设计器能力**
  - 审批人查看审批时，直接看到：变更前后结构对比、字段差异高亮
  - 可直接在审批页执行：SQL语法检查、性能风险评估
  - 审批意见直接同步回设计器，提交人可快速调整重提

- **执行结果反哺设计器**
  - 变更执行成功后，设计器自动刷新为最新表结构
  - 执行失败的错误信息直接在设计器中显示，方便定位修改
  - 变更历史与设计版本一一对应，可追溯回滚

#### 2.4.1 审批流程引擎
- **流程配置**
  - 可视化流程设计器
  - 多级审批节点配置
  - 条件分支（根据变更类型、影响范围、库类型等）
  - 会签/或签模式
  - 审批超时自动处理
  
- **工单管理**
  - 工单创建、提交、撤回、催办
  - 审批意见记录
  - 工单状态实时追踪
  - 工单统计报表
  - 工单模板复用

#### 2.4.2 团队协作
- **工作空间**
  - 项目级隔离
  - 成员角色管理
  - 项目资源配额

## 5. 非功能性需求

### 5.1 性能要求
- SQL编辑器响应时间 < 100ms
- 查询结果首屏展示 < 1s
- 工单审批流转 < 5s
- 系统支持并发用户数 > 1000

### 5.2 可用性要求
- 系统可用性 99.9%
- 故障恢复时间 < 30分钟
- 数据备份恢复点目标（RPO）< 1小时

### 5.3 安全性要求
- 所有操作可审计追溯
- 敏感数据不落盘明文
- 权限越权自动拦截
- SQL注入攻击防护

### 5.4 易用性要求
- 简洁直观的操作界面
- 完善的引导与帮助文档
- 快捷键支持

### 6.2 V2.0 - 标准版（DDL体验重构重点）

#### 🎯 核心目标：实现DDL变更的一体化体验
**对标阿里云DMS企业级体验，解决"多页面跳转、SQL复制粘贴、上下文丢失"痛点**

- **DDL变更一体化工作台**
  - 表结构设计与DDL部署流水线深度整合
  - 同页面完成：选库→选表→可视化修改→SQL预览→风险评估→提交审批→执行监控
  - 彻底消除：复制SQL、重复选择数据库等低效操作
- **无锁变更（Online DDL）**
  - 内置于工作台的"高级执行选项"
  - 支持：MySQL原生Online DDL、PT-OSC工具
  - 实时进度监控、速度控制、暂停/终止

## 8. 产品菜单结构优化建议

### 8.1 推荐菜单结构（对标阿里云DMS）

```
数据开发与变更
├── SQL查询窗口          # 单独的查询与数据分析场景
├── DDL变更工作台        # 核心整合：表结构设计 + 无锁变更 + 执行监控
│   ├── 新建表
│   ├── 修改已有表
│   └── 变更记录查询
├── 数据变更工单          # DML数据变更（INSERT/UPDATE/DELETE）
└── 数据导入导出

数据资产管理
├── 元数据检索
├── 数据字典
└── 数据质量

安全与审计
├── 敏感数据管理
├── 权限管理
└── 操作审计

系统管理
├── 数据库实例
├── 用户管理
└── 系统设置
```

### 8.2 菜单优化说明

| 优化项 | 原设计问题 | 优化后收益 |
|--------|-----------|-----------|
| **DDL变更工作台独立** | 表结构设计、无锁变更、工单审批分散在3个菜单 | 一个入口完成所有DDL相关操作 |
| **按场景组织** | 按技术领域切割菜单（设计 vs 执行 vs 审批） | 按用户真实工作流组织功能 |
| **减少跳转** | 平均每个DDL需跳转5-7次页面 | 同一页面完成整个DDL生命周期 |

> **阿里云DMS设计启示：好的产品不是功能的堆砌，而是场景的编排。让用户在对的地方做对的事。**
