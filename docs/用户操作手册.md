## 1. 系统概述

**DataOps DMS** 是一站式数据管理服务平台，提供多数据库统一管理、SQL查询、数据变更工单、审批流程、数据脱敏、敏感数据保护、权限管控等企业级功能。

### 1.1 技术架构

| 层级 | 技术栈 |
|------|--------|
| **前端** | React 18 + TypeScript + Ant Design 5 + Monaco Editor + Zustand |
| **后端** | Spring Boot 3.2 + Java 17 + MyBatis-Plus |
| **数据库** | MySQL 8.0（系统元数据库） |
| **认证** | JWT + Spring Security |
| **工作流** | Flowable 7.0 BPMN |
| **SQL解析** | JSqlParser 4.7 |
| **多数据库** | MySQL、PostgreSQL、Oracle、SQL Server、OceanBase、达梦、人大金仓等 |

### 1.2 功能模块一览

| 模块 | 说明 |
|------|------|
| **工作台** | 系统概览、统计指标、最近工单 |
| **SQL查询** | Monaco SQL编辑器，智能补全，结果可视化 |
| **数据库管理** | 多类型数据库实例接入、连接测试、健康检查 |
| **结构设计** | DDL可视化设计，ALTER TABLE自动生成，跨环境流转 |
| **数据导入** | CSV等格式数据批量导入 |
| **性能监控** | 数据库状态、慢查询、锁信息、表统计 |
| **元数据管理** | 实例→库→表→列四级元数据浏览 |
| **数据变更工单** | 工单创建、审批、执行、回滚全流程 |
| **审计日志** | 全链路操作记录，风险等级评估 |
| **数据脱敏** | 9种脱敏算法，动态脱敏规则配置 |
| **数据质量** | 质量规则管理 + 检查结果 |
| **权限管理** | 用户 → 角色 → 权限码 RBAC 体系 |
| **资源Owner** | 实例/库/表三级资源Owner分配 |
| **访问控制** | 数据库资源搜索可见性开关 |
| **敏感数据** | 敏感列标记、脱敏规则关联 |
| **权限申请** | 用户自主权限申请 + 审批流程 |
| **行级管控** | SQL WHERE条件过滤数据行 |
| **用户管理** | 用户CRUD、角色分配、密码重置 |
| **通知管理** | 邮件/钉钉/企微/Webhook多渠道通知 |
| **系统设置** | 系统参数全局配置 |

## 2. 快速入门

### 2.1 登录系统

1. 打开浏览器访问 `http://localhost:3000`
2. 输入管理员账号密码登录

> **默认管理员账号**: `admin` / `admin123`

### 2.2 界面布局

```
┌──────────────────────────────────────────────┐
│  Header: 系统标题 + 用户头像/通知/退出         │
├──────────┬───────────────────────────────────┤
│          │                                   │
```

## 4. SQL 查询窗口

**权限**: `sql:query`  
**路由**: `/sql`

### 4.1 功能说明

提供专业级 SQL 查询编辑器，支持多数据库实例查询。

### 4.2 操作步骤

1. 点击左侧菜单「SQL查询」
2. 在顶部下拉框中选择**数据库实例**
3. （可选）选择目标**数据库/Schema**
4. 在 Monaco Editor 中输入 SQL 语句
5. 点击「执行SQL」按钮（或按 `Ctrl+Enter`）
6. 在底部结果面板查看查询结果

### 4.3 编辑器特性

| 功能 | 说明 |
|------|------|
| **语法高亮** | SQL 关键字、表名、字段名按颜色区分 |
| **智能补全** | 输入时自动提示表名、字段名、SQL关键字 |
| **多语句执行** | 支持多条 SQL 之间用分号分隔执行 |
| **结果导出** | 查询结果支持导出为 CSV/Excel |
| **查询历史** | 自动记录查询历史，方便追溯复用 |

## 6. 结构设计（DDL工作台）

**权限**: `ddl:workbench`  
**路由**: `/schema-designer`

### 6.1 功能说明

提供表结构可视化设计工具，支持两种模式：

| 模式 | 说明 |
|------|------|
| **修改已有表** | 选择现有表，可视化修改字段并自动生成 ALTER TABLE SQL |
| **全新设计** | 从零创建新表结构 |

### 6.2 修改表结构操作步骤

1. 点击「结构设计」菜单
2. 选择「修改已有表」Tab
3. 选择**数据库实例** → **数据库** → **表**
4. 系统自动加载当前表结构，以可视化列表展示所有字段
5. 对字段进行增/删/改操作：
   - **新增字段**: 点击「添加字段」，选择数据类型、长度、默认值等
   - **修改字段**: 直接编辑字段属性
   - **删除字段**: 点击字段旁的删除按钮
6. 系统自动对比原结构，生成对应的 `ALTER TABLE` SQL
7. 选择执行方式：
   - **直接执行**: 立即在目标数据库执行 DDL
   - **提交工单**: 创建 DDL 变更工单走审批流程
   - **环境流转**: 通过流水线从开发环境 → 集成环境 → 生产环境

### 6.3 风险评估

系统自动对变更操作进行风险评估：

| 风险级别 | 场景 |
|----------|------|
| 🔴 高风险 | 类型变更、长度缩小、可空改非空 |
| 🟡 中风险 | 新增非空列、修改默认值 |
| 🟢 低风险 | 新增可空列、修改注释 |

### 6.4 环境流水线

```
开发环境 ──→ 集成环境 ──→ 生产环境
  (Step1)     (Step2)      (Step3)
```

- 点击进度条可切换 Tab 查看各阶段状态
- 支持回退到上一环境
- 每个环境可独立选择执行方式

## 10. 数据变更工单

**权限**: `ticket:create`（创建）/ `ticket:approve`（审批）/ `ticket:rollback`（回滚）  
**路由**: `/tickets`

### 10.1 功能说明

数据变更工单是系统核心功能，提供安全的数据库变更全流程管理：

```
创建工单 → SQL审核 → 审批 → 自动备份 → 执行 → 回滚（可选）
```

### 10.2 工单统计区

页面顶部显示工单统计卡片：

| 统计项 | 说明 |
|--------|------|
| 待审批 | 等待审批的工单数 |
| 执行中 | 正在执行的工单数 |
| 已完成 | 已成功执行的工单数 |
| 无锁变更 | 通过无锁分批执行的工单数 |

### 10.3 创建工单步骤

1. 点击「创建工单」按钮
2. 填写工单信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 工单标题 | 简要描述变更内容 | `修改管理员昵称` |
| 数据库实例 | 选择目标实例 | `homeplus-dev` |
| 数据库 | 选择目标数据库 | `dataops` |
| 变更类型 | DDL/DML | `DML` |
| 优先级 | 普通/紧急/低 | `普通` |
| SQL内容 | 待执行的SQL语句 | `update sys_user set nickname='系统管理员' where id=1` |
| 变更说明 | 变更原因和背景 | `需要统一管理员显示名称` |

3. 点击「SQL审核」检查 SQL 语句规范性和风险
4. 对于 DML 变更，可启用「无锁变更」模式：
   - 设置批次大小（100-5000行/批）
   - 设置执行间隔（0-500ms）
   - 系统自动评估风险等级和预估影响行数
5. 点击「提交」创建工单

### 10.4 SQL 审核规则

| 风险等级 | 检查项 | 说明 |
|----------|--------|------|
| 🔴 **高风险** | DROP / TRUNCATE | 检测表删除操作，禁止执行 |
| 🔴 **高风险** | DELETE/UPDATE 无WHERE | 全表数据变更风险 |
| 🟡 **中风险** | SELECT * | 全列查询性能问题 |
| 🟡 **中风险** | LIKE '%xxx' | 前缀通配符导致索引失效 |
| 🟢 **建议** | OR 条件过多 | 可能导致索引失效 |
| 🟢 **建议** | 无 LIMIT | 大结果集风险 |

### 10.5 工单审批

1. 拥有 `ticket:approve` 权限的用户在工单列表中可看到「待审批」状态的工单
2. 点击「审批」按钮
3. 查看工单详情和审核结果
4. 选择「通过」或「拒绝」
   - **通过**: 工单进入执行阶段
   - **拒绝**: 填写拒绝原因，工单状态变为已拒绝

### 10.6 工单执行与回滚

- **审批通过后**，系统自动执行以下流程：
  1. 解析SQL类型（INSERT/UPDATE/DELETE/DDL）
  2. 查询将被变更的原始数据
  3. **自动生成回滚SQL**（DELETE→INSERT, UPDATE→反向UPDATE）
  4. 保存备份记录
  5. 执行SQL变更
  6. 工单状态变为「已完成」

- **一键回滚**：已完成的工单可点击「回滚」按钮，系统使用自动生成的回滚SQL恢复数据

### 10.7 工单状态流转

```
草稿 → 待审批 → 审批中 → 待执行 → 执行中 → 已完成
  ↓       ↓                          ↓
 取消    拒绝                       失败 → 回滚中 → 已回滚
```

## 15. 资源 Owner 管理

**权限**: `owner:manage`  
**路由**: `/resource-owners`

### 15.1 功能说明

为数据库资源分配 Owner（负责人），Owner 可对该资源进行二次授权。

### 15.2 资源层级

| 层级 | 说明 |
|------|------|
| **实例 Owner** | 自动成为该实例下所有库的 Owner |
| **库 Owner** | 自动成为该库下所有表的 Owner |
| **表 Owner** | 负责单张表的管理 |

### 15.3 操作步骤

1. 左侧资源树浏览：点击展开节点（实例 → 数据库 → 表）
2. 点击某个资源节点，右侧展示该资源的 Owner 列表
3. 添加 Owner：
   - 在搜索框中输入用户名（至少2个字符触发搜索）
   - 选择目标用户
   - 点击「分配为Owner」
4. 移除 Owner：在列表中点击「移除」按钮

## 16. 访问控制

**权限**: `access:manage`  
**路由**: `/access-control`

### 16.1 功能说明

控制数据库资源对用户的**搜索可见性**。开启后，只有已获授权的用户才能搜索和访问该资源。

### 16.2 控制类型

| 类型 | 效果 |
|------|------|
| **实例访问控制** | 未授权用户无法搜索该实例 |
| **数据库访问控制** | 未授权用户无法搜索该数据库 |

### 16.3 操作步骤

1. 查看已有规则列表（资源类型、资源名称、启用状态）
2. 点击「添加规则」
3. 选择资源类型（实例/数据库）
4. 选择具体资源
5. 使用 Switch 开关启用/禁用访问控制

## 17. 敏感数据管理

**权限**: `sensitive:view`  
**路由**: `/sensitive-data`

### 17.1 功能说明

标记敏感数据列并关联脱敏规则，实现敏感数据的自动识别和保护。

### 17.2 两个 Tab

**脱敏规则**：查看系统内置脱敏规则列表（规则名、脱敏类型、脱敏模式、效果示例）

**敏感列标记**：
1. 左侧浏览数据库对象树（实例→表→列）
2. 点击某个列，选择「标记为敏感列」
3. 配置：
   - **敏感级别**: L1（低敏感）→ L4（极高敏感）
   - **分类标签**: 如"手机号"、"身份证"、"银行卡"
   - **脱敏规则**: 关联已配置的脱敏规则
4. 右侧面板查看已标记的敏感列列表
5. 支持取消标记操作

## 18. 权限申请

**权限**: 所有登录用户可访问（申请）；`ticket:approve`（审批）  
**路由**: `/permission-requests`

### 18.1 功能说明

用户可自主申请数据资源权限，无需管理员手动分配。

### 18.2 三个 Tab

**我的申请**：
- 查看历史申请记录
- 显示申请时间、资源类型、申请权限、状态、审批人
- 未处理(pending)的申请可撤销

**新建申请**：
1. 选择资源类型（实例/数据库/表/字段）
2. 填写资源ID
3. 选择申请权限：
   - `sql:query` - 数据查询
   - `sql:update` - 数据变更
   - `sql:export` - 数据导出
   - `sql:ddl` - 结构变更
4. 填写申请原因
5. 提交申请

**待审批**（仅审批人可见）：
- 查看待审批申请列表
- 点击「通过」或「拒绝」
- 拒绝需填写审批意见

## 19. 行级管控

**权限**: `access:manage`  
**路由**: `/row-controls`

### 19.1 功能说明

实现细粒度的数据行权限控制。通过 SQL WHERE 过滤条件，限制用户只能访问特定的数据行。

### 19.2 典型场景

```
开发者A 只能查询 department='研发部' 的数据行
数据分析师B 只能查看 status='active' 的记录
```

### 19.3 操作步骤

**创建规则**：
1. 点击「新建规则」
2. 配置：
   - 规则名称（如"研发部数据限制"）
   - 数据库实例 + 数据库 + 目标表
   - WHERE过滤条件（如 `department='研发部' AND status='active'`）
   - 条件描述（说明限制意图）
   - 目标用户ID（逗号分隔多个用户）
   - 目标角色ID（逗号分隔多个角色）
   - 优先级（数字越小优先越高）

**管理规则**：
- 规则列表展示所有配置
- Switch 开关启用/停用规则
- 支持编辑和删除规则

## 24. 常见问题

### 24.1 如何添加新的数据库实例？

参见 [第5章 数据库实例管理](#5-数据库实例管理)。需要拥有 `database:manage` 权限。

### 24.2 工单创建后如何审批？

拥有 `ticket:approve` 权限的用户：
1. 进入「数据变更工单」页面
2. 在待审批列表中点击「审批」
3. 查看 SQL 审核结果和变更详情
4. 选择「通过」或「拒绝」

### 24.3 如何给用户分配权限？

两种方式：
- **角色分配**: 在「用户管理」页面对用户分配角色（如 dba、developer）
- **直接授权**: 使用管理员账号在「权限管理」中直接授予用户权限

### 24.4 数据变更如何回滚？

对于已完成的 DML 工单：
1. 进入「数据变更工单」页面
2. 找到状态为「已完成」的工单
3. 点击「回滚」按钮
4. 系统自动执行回滚 SQL，恢复变更前的数据

> 注意：DDL 类变更（如 DROP TABLE）无法自动回滚，非无锁变更工单才支持回滚。

### 24.5 什么是无锁变更？

无锁变更是将大批量 DML 操作拆分为小批次执行，避免长时间锁表：
- 设置批次大小（100～5000行/批）
- 设置执行间隔（0～500ms）
- 系统自动预估总批次和耗时
- 适合数据量较大的 UPDATE/DELETE 操作

### 24.6 如何标记敏感数据列？

参见 [第17章 敏感数据管理](#17-敏感数据管理)。需拥有 `sensitive:view` 或 `sensitive:manage` 权限。

### 24.7 忘记了管理员密码怎么办？

联系系统管理员使用数据库直接修改 `sys_user` 表中的密码（密码经过 BCrypt 加密存储）。

### 24.8 如何创建新用户？

1. 使用管理员账号登录
2. 进入「用户管理」页面
3. 点击「创建用户」
4. 填写用户名、密码、昵称、邮箱
5. 为用户分配角色（可选）

### 24.9 服务如何启动与停止？

**启动**：
```bash
# 后端
cd backend && mvn spring-boot:run

# 前端
cd frontend && npm run dev
```

**停止**：
```bash
# 后端：Ctrl+C 中断进程
# 或使用 taskkill（Windows）
taskkill /f /im java.exe
```

### 24.10 数据库连接失败如何排查？

1. 确认目标数据库服务是否正常运行
2. 检查数据库实例配置中的主机地址、端口、用户名、密码
3. 确认网络防火墙是否允许连接
4. 使用「测试连接」功能验证连通性
5. 查看后端日志 `backend.log` 查看详细错误信息
